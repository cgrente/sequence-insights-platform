# Security Policy

## Supported Versions

This project does not currently maintain multiple supported versions.
Security fixes will be applied to the latest release on the `main` branch.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **open a GitHub issue**
and clearly label it as **security-related**.

When possible, avoid including sensitive details such as secrets,
tokens, or private data in the initial report.

Please include:
- a clear description of the issue
- steps to reproduce
- potential impact
- any suggested remediation

---

## Response Policy

Security-related issues will be reviewed as soon as reasonably possible.

We aim to acknowledge reports within **72 hours** and provide
a fix or mitigation based on severity and scope.

---

## Scope

This policy applies to:
- API authentication
- request validation
- file handling
- data storage
- dependency vulnerabilities

---

Thank you for helping keep the project secure.
